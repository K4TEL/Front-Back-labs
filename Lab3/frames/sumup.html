Висновки: було набуто практичні навички роботи верстки сторінок засобами CSS, верстки сторінок засобами CSS та FLEXBOX, верстки на основі плаваючих елементів, з’ясовано переваги та недоліки типів макетів веб-сторінок.

Порівняння розмітки за допомогою таблиці та плаваючих елементів: CSS Grid і Flexbox — це моделі макетів, які мають схожість і можуть використовуватися разом. Ключова відмінність полягає в тому, що CSS Grid можна використовувати для створення двовимірних макетів, тоді як Flexbox можна використовувати лише для створення одновимірних макетів. Це означає, що ви можете розміщувати компоненти вздовж осі X і Y в CSS Grid і тільки по одній осі в Flexbox.

1. Розмірність і гнучкість: Flexbox забезпечує кращий контроль над вирівнюванням і розподілом простору між елементами. Будучи одновимірним, Flexbox має справу лише зі стовпцями або рядками. Сітка має можливості двовимірного макета, що дозволяє змінювати ширину як одиницю довжини. Це компенсує обмеження у Flex.

2. Вирівнювання: Flex Direction дозволяє розробникам вирівнювати елементи по вертикалі або горизонталі, що використовується, коли розробники створюють і повертають рядки або стовпці. CSS Grid розгортає дробові одиниці вимірювання для плавності сітки та функції автоматичного введення ключових слів для автоматичного налаштування стовпців або рядків.

3. Управління предметами Flex Container є батьківським елементом, а Flex Item представляє дочірні елементи. Flex Container може забезпечити збалансоване представлення, регулюючи розміри елемента. Це дозволяє розробникам розробляти дизайн для коливань розмірів екрана. Grid підтримує як неявне, так і явне розміщення вмісту. Його вбудована автоматизація дозволяє автоматично розширювати рядки та копіювати значення в нове створення з попереднього елемента.
